# Running the Demos

Quick guide to test the filesystem-free Move compiler in your browser.

## Prerequisites

- Rust 1.75+ with `wasm32-unknown-unknown` target
- wasm-pack
- Python 3 (for serving files) or any static file server

## Quick Start

### 1. Build the WASM Package

```bash
cd aptos-move/move-compiler-wasm

# Build for web (optimized)
wasm-pack build --target web --release
```

This creates a `pkg/` directory with:
- `move_compiler_wasm.js` - JavaScript bindings
- `move_compiler_wasm_bg.wasm` - WASM binary (~2-3MB)
- `move_compiler_wasm.d.ts` - TypeScript definitions

### 2. Start a Local Server

```bash
# From the repository root
python3 -m http.server 8000

# Or use any other static file server
# npx http-server -p 8000
# php -S localhost:8000
```

### 3. Open the Demos

Navigate to http://localhost:8000/aptos-move/move-compiler-wasm/demo/

Available demos:

#### 📋 Demo Landing Page
**URL:** http://localhost:8000/aptos-move/move-compiler-wasm/demo/demos.html

Overview of all available demos with links and instructions.

#### 🧪 E2E Test Suite
**URL:** http://localhost:8000/aptos-move/move-compiler-wasm/demo/test-e2e.html

Comprehensive end-to-end tests:
- Click "Run All Tests" to execute the full test suite
- Tests include:
  - Simple module compilation
  - Multiple functions
  - Structs and generics
  - Syntax error handling
  - Type error detection
  - Vector operations
  - Script compilation

**Expected Results:**
- All tests should pass
- Compilation times: 10-200ms per test
- Real bytecode generated and verified

#### ⚡ Performance Benchmarks
**URL:** http://localhost:8000/aptos-move/move-compiler-wasm/demo/benchmark.html

Measures compilation performance:
- Select complexity level (small/medium/large/all)
- Set number of iterations (default: 10)
- Click "Run Benchmarks"

**Metrics:**
- Compilation time (min/avg/max/median)
- Throughput (lines of code per second)
- Bytecode size
- Visual charts and detailed tables

**Expected Performance:**
- Small (28 LOC): ~50-100ms, ~3000 LOC/s
- Medium (142 LOC): ~100-200ms, ~2500 LOC/s
- Large (240 LOC): ~150-300ms, ~2000 LOC/s

#### 💻 Interactive IDE
**URL:** http://localhost:8000/aptos-move/move-compiler-wasm/demo/index.html

Write and compile Move code interactively:
- Edit Move source code
- Click "Compile" for instant feedback
- See compilation results or errors
- View generated bytecode

## Troubleshooting

### WASM Module Not Found
**Error:** `Failed to fetch ../pkg/move_compiler_wasm_bg.wasm`

**Solution:** Make sure you've built the WASM package and are serving from the correct directory.

```bash
# Build if not done
cd aptos-move/move-compiler-wasm
wasm-pack build --target web --release

# Serve from repo root
cd /opt/git/aptos-core
python3 -m http.server 8000
```

### CORS Errors
**Error:** `CORS policy: No 'Access-Control-Allow-Origin' header`

**Solution:** Use a proper HTTP server (not file:// protocol). The `python3 -m http.server` command above handles this correctly.

### Slow Compilation
**Issue:** First compilation takes a long time

**Explanation:** WASM initialization and compilation happens on first use. Subsequent compilations are much faster due to JIT optimization.

### Memory Issues
**Error:** `Out of memory` or browser tab crashes

**Solution:**
- Use a 64-bit browser
- Close other tabs to free memory
- Reduce benchmark iterations
- Test with smaller code samples first

## Performance Tips

### Browser Developer Tools
Open browser DevTools (F12) to see:
- Console logs with compilation times
- Network tab showing WASM loading
- Performance profiling

### Optimization Levels

**Dev Build** (faster compilation, larger binary):
```bash
wasm-pack build --target web --dev
```

**Release Build** (slower compilation, smaller binary):
```bash
wasm-pack build --target web --release
```

**Further Optimization**:
```bash
wasm-pack build --target web --release
wasm-opt -Oz pkg/move_compiler_wasm_bg.wasm -o pkg/move_compiler_wasm_bg.wasm
```

## What to Expect

### First Load
- WASM download: ~2-3MB (gzipped: ~600KB)
- Initialization: ~100-500ms
- Total time to ready: ~1-2 seconds

### Compilation Performance
- Simple modules: 50-100ms
- Medium complexity: 100-200ms
- Large modules: 150-300ms
- Subsequent compilations: Often faster due to JIT
